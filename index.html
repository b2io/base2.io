What We Do

Our Process

Requirements Gathering

The first step to solving a problem is understanding it. We put a huge emphasis on understanding our clients’ work. For us, requirements gathering isn’t just a phase at the beginning of the project, it is an ongoing process. We are always working to better understand our clients’ business needs and converting those needs into user stories and project milestones.

UX/UI

Great code is useless without great user interface design. Base Two delivers both.

We start by creating user stories to get a firm understanding of how your average user works and to tease out any hidden interaction requirements. Then we audit your content, optimizing it for web and mobile interfaces and organizing it so your users can navigate intuitively and effectively.

The resulting Information Architecture forms the foundation for the structure of the project. We use iterative wireframes and Rapid Prototyping to mock up and then fine-tune the user flow. Once the application flow is just right, we skin the wireframes, producing a beautiful, usable design.

Agile Development

At Base Two, we manage projects differently. We follow a software development discipline known as the Agile Method, a philosophy acknowledges that requirements and solutions evolve throughout the project lifecycle. With Agile, we divide the project scope into many small milestones with clear, concise goals. The result? Our customers begin receiving deliverables in the first weeks of development, get an earlier return on investment, and are never out of the loop.

Test Driven Development

TDD is a software development methodology we use to ensure that our software meets user requirements, and that it continues to meet the requirements as new features are introduced. For each new feature, we write an initially failing test, develop the feature until the test passes and then refactor the code for optimization and maintenance. With TDD, every feature has associated tests explaining exactly how the code should function, so the verification TDD provides also serves as documentation for the application.

Clients

ScopeBuilder

We worked with AEP to express one of their bread-and-butter engineering processes: scoping and ordering of equipment for transmission projects. During discovery, AEP learned that they would benefit substantially from a reporting system that could satisfy queries like: How many Drop-In Control Modules are they installing this summer? Which engineer or engineering group most frequently places costly, non-standard orders?

We built ScopeBuilder with the modern client-side Javascript framework, Backbone.js, on an ASP .NET back-end. By deploying weekly builds to a cloud-service using Heroku and Azure, we eliminated the need for client-provided infrastructure support, getting each week’s deliverable directly into the client’s hands.

TSV

AEP needed help. Engineers were completing and filing maintenance reports manually, effectively forfeiting all opportunity to gain institutional knowledge from their projects. Working with PCE team to describe every facet of their process, we transformed their manual reporting process into a modern web application on a Ruby on Rails server.

We weren’t thrown when the client’s technological needs suddenly changed from Rails to ASP.NET during development. Because we built the web application to be platform agnostic, we could seamlessly integrate it with any client-side services technology.

This means that while our ASP.NET expert was building a brand-new set of services for the application, our UI Developer was free to work on implementing new features. For our team and our client, what could have easily become a major roadblock was nothing more than a minor bump in the road.

Nature’s One Mobile

Nature’s One, the leader in organic pediatric medical nutrition, came to Base Two to build an eCommerce solution and a mobile application after a previous vendor failed to deliver. We used Magento (built in PHP with the Zend Framework) for the eCommerce back-end and jQuery Mobile with Phonegap to deploy a gorgeous, streamlined mobile store to Android, iOS, and Windows Phone from a single codebase.

KJM Products Prototype

For a startup seeking venture capital, money is tight and timing is crucial. KJM Products approached us with an idea for a product that we estimated would take 6 weeks to implement. That didn’t work for our client--they needed a working version sooner. By working with KLM to identify a MVP that could be presented to stakeholders sooner, we whittled the project estimate down from 6 weeks to 2 weeks. The application was written in web languages, styled with JQuery Mobile and compiled for iOS and Android through PhoneGap.
